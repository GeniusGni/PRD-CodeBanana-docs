### Super Organization and the CodeBanana Product Philosophy

An excerpt from Mobvoi CEO’s Super Organization sharing session, introducing the product background, core ideas, and design philosophy behind CodeBanana.

---

## Why Build CodeBanana?

There are two kinds of work inside a company, yet they have long been handled with two disconnected sets of tools:

| Tool Type | Representative Products | Purpose |
|----------|----------|------|
| 💬 Communication & Management | Feishu, DingTalk | Meetings, documentation, coordination |
| ⚙️ Task Execution | Cursor, Claude Code | Coding, design, execution |

**The problem:** Meetings happen in Feishu, notes are taken there, but the actual execution happens in another tool—so no one really knows what happened in between.

**The solution:** CodeBanana brings both into one place—communication and execution happen together, with context synced in real time, so decisions can turn directly into action.

---

## Three Core Characteristics of CodeBanana

- 🤖 **Agent-first** — The system is centered around Agents. Humans and Agents work together, rather than humans controlling everything alone.
- 🏢 **Built for organizations** — Supports multi-user collaboration, clear permission management, and isolated file systems across projects for security and efficiency.
- ⚡ **Communication and execution in one** — Integrates group chat, Agent workspace, and file system so communication and task handling flow seamlessly together.

---

## Centered Around the “Project”

Each project is a three-in-one entity:

- 🤖 **An Agent** — Every project comes with its own dedicated Agent, with independent execution capabilities to complete tasks, write code, and handle files.
- 💬 **A group chat** — At the same time, it is also a real-time collaboration space where team members and the Agent communicate within the same context.
- 🗂️ **A Workspace** — An independent file system and virtual machine with secure data isolation, where the Agent can directly read, write, run, and deploy.

---

## System Architecture

| Component | Description |
|------|------|
| 🟣 **Team Agent** | A shared Agent. Only one person can operate it at a time, keeping collaboration focused. |
| 🔵 **Private Agent** | A personal Agent that runs in an independent private space. |
| 🟠 **Discussion** | The human communication area, where team discussions and decisions are synced in real time. |
| 🟢 **Workspace** | Files and virtual machines, forming the foundational layer of the three-in-one collaboration environment. |

---

## Built for Collaboration

- 👥 **Real-time multi-user chat** — Team members can communicate freely within the same project space, while the Agent is always present and can be called at any time.
- ⚡ **Turn chat directly into tasks** — Conversations between people can be forwarded to the Agent with one click for execution; communication becomes instruction.
- 📤 **Easy flow of files and messages** — Files and messages inside the system can be quickly forwarded, quoted, and shared, so information is no longer isolated.
- 🛍️ **Skill publishing and sharing** — Custom capabilities can be published to the Skill Market and reused across projects and teams, allowing organizational knowledge to accumulate.

---

## Separating Human and Agent Messages

Agent work messages are frequent and dense. If they are mixed with human conversations, no one will want to read them.

| Tab | Content |
|-----|------|
| 💬 **Discussion** | Human-to-human communication — discussion, decisions, collaboration |
| 🤖 **Agent** | Agent execution process — tasks, code, results |

Two separate tabs with separate notifications ensure that human messages do not get buried.

---

## Let Agents Work Proactively (Cron Jobs)

Agents do not only act when you ask them to—through Cron Jobs, they can execute tasks automatically on schedule:

- ⏰ **Scheduled triggers** — Set a time or recurring cycle, and the Agent wakes up automatically to work: daily reports, periodic checks, scheduled reminders.
- 🔁 **Event-driven execution** — File changes, incoming messages, condition-based triggers—the Agent listens for changes and responds immediately.
- 🤝 **From passive to proactive** — The Agent is no longer just a tool, but a true collaborator that actively helps drive organizational goals forward.

---

## Skill Market — Let Great Methods Spread Across the Company

- 📋 **Workflow templates for different roles** — Skills capture the working methods of each function and become effective as soon as they are installed.
- ⬆️ **One person builds it, the whole company uses it** — When someone discovers an efficient way of working, they can package it as a Skill and publish it to the market for everyone else to use.
- 🚀 **Best practices spread virally** — Others can install and reuse them with one click, allowing organizational best practices to truly circulate.

---

## Enterprise Administration Capabilities

- 📊 **Dashboard for global insights** — Clearly understand enterprise AI model usage: which departments are using it well, and who the standout high-performers are.
- 🏗️ **Organizational structure configuration** — Configure teams and projects based on the company’s real structure, with clear management hierarchy.
- 🔐 **Fine-grained permission control** — Configure and restrict access for each individual, ensuring data security, compliance, and control.

## How CodeBanana Differs from Cursor / Claude Code

| Dimension | Cursor / Claude Code | CodeBanana |
|------|----------------------|------------|
| Positioning | Personal development tool | Team collaboration platform |
| Communication | None, requires switching to Feishu/DingTalk | Built-in group chat, communication and execution unified |
| Collaboration | No shared Agent support | Team Agent supports team sharing |
| Context | Local / individual | Cloud-based, shared team context |
| Organization management | None | Supports org structure and permission control |
| Skill reuse | None | Skill Market for cross-project reuse |

## Core Philosophy

> An Agent is not just a tool—it is an extension of organizational capability.  
> CodeBanana aims to help every organization evolve into a super organization, using AI to multiply the productivity of every individual.
