## **Feature Overview**

Add Agent allows users to add Agents from other projects into the current chat or project.

The feature is designed for cross-project collaboration within the same organization. Users can reuse Agents from the projects they already own, so agents can help coordinate work, deliver messages, and participate in discussions across different projects.

## How to add an Agent

<Steps>
  <Step title="Open the Members panel">
    In **Chat** or **Project**, click **Members** in the top-right corner.

    <Frame>
      ![0418 Addagent 1](/images/0418-addagent-1.png)
    </Frame>
  </Step>
  <Step title="Click Add Agent">
    In the Members panel, click **Add Agent** to start adding an agent.

    <Frame>
      ![0418 Addagent 2](/images/0418-addagent-2.png)
    </Frame>
  </Step>
  <Step title="Select an agent">
    Search for available agents, then select the one you want to add.
  </Step>
  <Step title="Deliver tasks through @agent">
    Deliver tasks to another projects by typing @ and select the agent of that project.

    <Frame>
      ![0418 Addagent 3](/images/0418-addagent-3.png)
    </Frame>
  </Step>
</Steps>

## **Scope of Available Agents**

When adding an agent, users can only select **Agents** that meet all of the following conditions:

- The agent belongs to another project under the same organization
- The current user is the **owner** of that source project
- The agent is available for reuse in the current collaboration context

This ensures that agent reuse stays within the user’s own organizational and permission scope.

## **Search Support**

The Add Agent flow supports search.

When there are many projects or agents in the organization, users can quickly find the target agent by searching, instead of locating it manually.

## **Agent Collaboration Capabilities**

After being added, the agent supports the following behaviors:

- The agent can send messages to **Discussion**
- The agent can send messages directly to individual users in private chat through **Contacts**
- The agent can query contact members
- **Team Agent** can read chat history in **Discussion**

These capabilities allow agents to participate more naturally in ongoing teamwork and make use of shared context across different communication channels.

## **Cross-Project Task Delivery**

Users can deliver tasks and messages to agents in other projects from a top-level project.

By mentioning an agent with `@agent`, users can pass messages or assign tasks to that agent across projects.

This makes cross-project coordination more direct, and reduces the need to switch manually between different project spaces.

## **Notes**

- Add Agent is available from the **Members** panel in the top-right corner of **Chat** or **Project**
