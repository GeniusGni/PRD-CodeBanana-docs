## Requests 与 Token

#### **什么是 Request**

**Request** 表示 Agent 进行一次输出。

- 每次 Agent 成功回复一次都会消耗一个 request
- 包括：
  - 代码生成
  - 问答
  - 任务执行

不同套餐通常会根据订阅档位，按月提供固定数量的 requests。

#### **什么是 Token**

**Token** 用于衡量模型实际消耗的计算量。

- 包括：
  - 输入内容（您的 prompt、上下文、文件）
  - 输出内容（Agent 回复、生成的代码）
- 任务越复杂、上下文越大，消耗的 token 通常也会越多

Token 用于计算 **按用量计费成本**，尤其适用于超出套餐额度后的使用部分。

#### **计费方式如何运作**

用量将基于 requests 和 tokens 共同计算：

1. **在套餐额度内** requests 会从您的月度配额中扣除
2. **超出套餐额度后** 计费可能会切换为 **按用量计费（基于 token）**
3. **Team Agent 与 Private Agent（此前称为 My agent）**
   - Team Agent 的使用费用由 **项目 Owner** 承担
   - Private Agent 的使用费用同样由 **项目 Owner** 承担

#### **其他说明**

- 使用更高级的模型或更大的上下文，可能会增加 token 消耗
- 某些功能（例如支持服务会话或增强能力）可能会对用量 적용 **倍数系数**
- 您可以在 **Usage dashboard** 中查看详细的用量信息
