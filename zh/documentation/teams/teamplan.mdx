## **什么是团队套餐**

团队套餐提供一种 **协作式使用模式**，其特点包括：

- 资源（requests、算力、usage limits）在 **组织层级** 统一管理
- 账单由 **组织 Owner** 统一支付

### **套餐结构**

团队套餐由两个主要部分组成：

**预付费（包含用量）**

- 每位成员每月可获得 **300 次 requests**
- 可根据使用需求选择 VM 配置，由 **组织管理员** 统一控制
- 费用会根据所选 VM 档位而变化（以席位费形式收取）

**后付费**

- 当包含的 requests 用尽后启用
- 额外用量将按实际消耗计费
- Usage limits 由 **组织管理员** 进行定义和控制

**升级行为**：

- 立即生效
- 差价将由 Owner 扣费（**仅适用于 VM 变更**）

**降级行为**：

- 剩余余额将保留在 Stripe 中（**仅适用于 VM 变更**），用于抵扣后续账单

#### **升级 / 降级的价格计算逻辑**

使用成本按以下 **两者中的较高值** 计算：

- **（套餐总价 / 总天数）× 已使用天数**，或
- **每次 request \$0.15**

剩余余额计算方式：

- 余额 = 已支付金额 − 已使用金额
- 升级费用 = 新价格 − 剩余余额

#### **成员使用体验**

启用团队套餐后：

成员需要由 **管理员手动添加**

添加成功后，成员将获得：

- 每月 300 次 requests
- 已分配的 VM 资源
- 已配置的 usage limits

当额度用尽时：

- 成员需要联系管理员提升配额

如果成员 **未被添加到团队套餐**：

- 其用量将按 **个人套餐** 进行计费

#### **计费机制**

**席位费**

- 激活团队套餐时，会先收取管理员席位费用
- 添加成员时，会向 Owner 收取对应席位费用

**用量计费**

- 在以下任一条件满足时自动扣费：
  - On-demand usage 达到 **\$2000**
  - 每满 **30 天**

## FAQs

团队套餐常见问题解答

<AccordionGroup>
  <Accordion title="为什么我在“Add Members”列表里找不到某个成员？">
    该成员尚未完成加入组织的申请。请让其登录 CodeBanana，前往 “Personal Center → Organization”，搜索您的组织名称并提交申请。待审批通过后，该成员才会出现在可添加列表中。
  </Accordion>

  <Accordion title="是否可以为不同成员分配不同的 VM 配置？">
    可以。在 Add Members 步骤中，“Per-Member Configuration” 区域支持为每位成员单独设置 VM 规格和 usage limits。
  </Accordion>

  <Accordion title="成员超出 usage limit 后会发生什么？">
    当 On-Demand limit 超出后，该成员将无法继续使用按量资源，直到管理员提升其额度。
  </Accordion>

  <Accordion title="30 天计费周期是固定日期，还是滚动周期？">
    其采用自团队套餐激活之日起开始计算的滚动 30 天周期，而非固定自然月日期。如果总用量先达到 \$2,000，则会提前触发扣费。
  </Accordion>

  <Accordion title="在哪里可以查看账单记录？">
    管理员可以在 Usage 页面查看全部账单历史以及各成员的详细用量信息。系统也会将账单通知发送至管理员的注册邮箱。
  </Accordion>
</AccordionGroup>
